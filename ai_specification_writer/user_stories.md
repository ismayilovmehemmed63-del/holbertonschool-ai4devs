# User Stories

### User Story 1
As a passenger, I want to search for nearby drivers so that I can find a quick ride.
**Acceptance Criteria**:
- User enters pickup and destination.
- System displays available drivers within 5 km.
- Ride can be booked in under 2 minutes.
**Priority**: MVP

---

### User Story 2
As a driver, I want to post my ride details so that passengers can find and book my car.
**Acceptance Criteria**:
- Driver enters route, departure time, and available seats.
- Ride is visible to matching passengers immediately.
- Driver receives booking notifications in real time.
**Priority**: MVP

---

### User Story 3
As a passenger, I want to pay for my ride in-app so that I do not need to carry cash.
**Acceptance Criteria**:
- User can add a credit/debit card or mobile payment method.
- Payment is processed automatically after ride completion.
- Receipt is sent to user's email.
**Priority**: MVP

---

### User Story 4
As a user, I want to verify my identity so that other users feel safe riding with me.
**Acceptance Criteria**:
- User uploads a government-issued ID during registration.
- System verifies document within 24 hours.
- Verified badge is displayed on user profile.
**Priority**: MVP

---

### User Story 5
As a passenger, I want to track my driver in real time so that I know when to be ready.
**Acceptance Criteria**:
- Passenger sees driver location on a live map after booking.
- Estimated arrival time updates every 30 seconds.
- Passenger receives a notification when driver is 2 minutes away.
**Priority**: High

---

### User Story 6
As a passenger, I want to rate my driver after the trip so that I can help maintain platform quality.
**Acceptance Criteria**:
- Rating prompt appears automatically after ride completion.
- User can give 1-5 stars and an optional comment.
- Driver's average rating is updated immediately.
**Priority**: High

---

### User Story 7
As an administrator, I want to monitor all rides so that I can ensure platform safety.
**Acceptance Criteria**:
- Admin dashboard shows all active and completed rides.
- Admin can suspend users who violate platform rules.
- Incident reports can be filed and tracked within the system.
**Priority**: High

---

### User Story 8
As a user, I want to add emergency contacts so that someone is notified if I am in danger.
**Acceptance Criteria**:
- User can add up to 3 emergency contacts in profile settings.
- Contacts receive an SMS with ride details at trip start.
- User can trigger an emergency alert with one tap during a ride.
**Priority**: MVP
